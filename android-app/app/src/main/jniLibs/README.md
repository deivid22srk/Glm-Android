# jniLibs

This directory holds "fake" native libraries that are actually standalone
Go binaries, disguised as `.so` files so that Android's package manager
extracts them with execute permission.

## Why?

Android 10+ (API 29+) blocks `execve()` on any file inside `filesDir`,
`cacheDir`, or `assets/` due to the SELinux rule:

```
neverallow ... app_data_file:file execute;
```

The only filesystem location with execute permission for an app is
`/data/app/<pkg>/lib/<abi>/` — where the system installs native libraries
declared in `jniLibs/`. Files there have the SELinux label `apk_data_file`
which allows execution.

So we **disguise the Go binary as a `.so` library**:

- File name: `lib<name>.so` (prefix `lib` + suffix `.so` are mandatory)
- File content: any valid PIE ELF (Android doesn't validate that it's
  actually a shared library — just an ELF executable)
- Manifest: `android:extractNativeLibs="true"`
- Gradle: `packaging { jniLibs { useLegacyPackaging = true } }`

At runtime, the app locates the binary via:
```kotlin
val binary = File(context.applicationInfo.nativeLibraryDir, "libglmproxy.so")
```

And starts it with `ProcessBuilder(binary.absolutePath).start()`.

## File: `arm64-v8a/libglmproxy.so`

The committed file is a **stub 64-byte ELF64 aarch64 header** — enough for
the Android Gradle Plugin to package it without errors during local dev
builds. It is **not** a functional Go binary.

The real binary is generated by the CI workflow (`.github/workflows/build.yml`)
which cross-compiles `cmd/server` with:

```bash
GOOS=android GOARCH=arm64 CGO_ENABLED=1 \
  CC=$NDK/toolchains/llvm/prebuilt/linux-x86_64/bin/aarch64-linux-android24-clang \
  go build -buildmode=pie -ldflags='-s -w' \
  -o android-app/app/src/main/jniLibs/arm64-v8a/libglmproxy.so \
  ./cmd/server
```

The CI overwrites the stub before building the APK. If you want to build
the APK locally with a real Go binary, run that same command locally (you
need the Android NDK installed and `CC` pointed at it).

## Adding more ABIs

To support 32-bit ARM or x86_64 emulators, drop additional binaries at:
- `armeabi-v7a/libglmproxy.so`
- `x86_64/libglmproxy.so`

And add the ABI names to `abiFilters` in `app/build.gradle.kts`.
